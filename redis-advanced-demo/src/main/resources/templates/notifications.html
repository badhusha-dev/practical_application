Redis Pub/Sub Demo Information
How it works:
  • Messages are published to Redis channel "notifications"
  • All connected clients automatically receive messages
  • Sessions are stored in Redis for distributed access
  • Real-time communication without polling
API Endpoints:
  • POST /notifications/publish - Send message
  • GET /notifications/messages - Get all messages
  • POST /notifications/clear - Clear messages
  • GET /notifications/subscribe - Subscribe to channel
